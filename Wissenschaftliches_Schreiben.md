# Richtlinien für wissenschaftliches Schreiben

## 0. Verbindlichkeit und Prüfmaßstab

Wissenschaftliche Arbeiten werden nicht nach „Schönheit“, sondern nach **Prüfbarkeit** bewertet: Eine Leserin muss nachvollziehen können, *was* Sie getan haben, *warum* Sie es so getan haben und *wie* Ihre Schlussfolgerungen aus Evidenz oder sauberer Herleitung folgen.

| MUSS                                                                                                                  | SOLL                                                                                                            |
| --------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Gute wissenschaftliche Praxis (korrekte Zitation, keine verdeckte Übernahme, saubere Trennung von Befund und Deutung) | Jede zentrale Aussage ist auf **Quelle**, **Daten/Evidenz** oder **explizite Herleitung** zurückführbar         |
| Nachvollziehbarkeit: Design, Daten/Material, Auswertungsschritte und Entscheidungslogik sind rekonstruierbar          | Unübliche Entscheidungen werden begründet (Warum diese Methode, warum dieses Material, warum dieser Zuschnitt?) |
| Konsistenz: Begriffe, Notation, Zitierstil und Format sind durchgängig einheitlich                                    | Prüfer*innen-Perspektive aktiv mitdenken („Kann man das prüfen? Kann man das replizieren?“)                     |

## 1. Abstract

Der Abstract ist die kondensierte Frage-Antwort-Klammer der Arbeit. Er muss ohne Vorwissen verständlich sein und zeigt in minimalem Raum, dass die Arbeit **zielgerichtet**, **methodisch** und **ergebnisorientiert** ist.

| MUSS                                                                                         | SOLL                                                                 |
| -------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| **Deutsch und Englisch** im selben Abschnitt, direkt nacheinander (**DE → EN**)              | Wort-Richtwerte nur zur Einhaltung „1 Seite“: DE 150–250, EN 150–250 |
| Beide Versionen **im Präsens**                                                               | Keine Zitate, keine Fußnoten                                         |
| Beide Versionen zusammen **maximal 1 Seite** (kein Seitenumbruch zwischen DE und EN)         | Abkürzungen nur, wenn unvermeidbar; bei erster Nennung ausschreiben  |
| Pro Version: Problem → Fragestellung/Ziel → Methode/Daten → Ergebnisse → Beitrag/Implikation | Keine Leersätze („Die Arbeit zeigt …“) ohne Ergebnis                 |
| Ergebnisse konkret (Zahlen/Relationen/Muster statt wertender Adjektive)                      | Limitationen nur, wenn methodisch entscheidend und sehr knapp        |

## 2. Struktur und roter Faden

Struktur ist ein **Begründungsgerüst**: Ein Abschnitt ist nur dann legitim, wenn er erkennbar zur Beantwortung der Fragestellung beiträgt. „Roter Faden“ bedeutet nicht, dass alles irgendwie zusammenhängt, sondern dass **jede Teilentscheidung begründbar** ist.

| MUSS                                                                                                                                                                                | SOLL                                                                                        |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| Standardgliederung (sofern fachlich nicht anders begründet): Einleitung → Theorie/Forschungsstand → Methodik → Ergebnisse → Diskussion → Fazit → Literaturverzeichnis → ggf. Anhang | Kapitelüberschriften informativ statt allgemein („Grundlagen“, „Theorie“)                   |
| Jedes Kapitel beantwortet: „Was trägt es zur Beantwortung der Fragestellung bei?“                                                                                                   | Pro Kapitel: Ziel (1–2 Sätze) → Kernpunkte → Mini-Fazit (1–3 Sätze) → Überleitung           |
| Ergebnisse und Diskussion strikt trennen (Befunddarstellung ≠ Deutung)                                                                                                              | Redundanzen vermeiden: keine vollständigen Ergebnisblöcke in früheren Kapiteln vorwegnehmen |

## 3. Fragestellung und Beitrag

Eine tragfähige Fragestellung ist so präzise, dass methodische Entscheidungen daraus **ableitbar** werden. Eine diffuse Frage erzeugt diffuse Methodik und diffuse Ergebnisse.

| MUSS                                                                                    | SOLL                                                                                     |
| --------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| Fragestellung als **ein** klarer Satz (Gegenstand + Kontext + Zielgröße)                | Beitragstyp benennen (theoretisch/methodisch/empirisch/praktisch)                        |
| Abgrenzung explizit formulieren („Untersucht wird … / Nicht untersucht wird …, weil …“) | Ergebnisform benennen (Modell, Typologie, Hypothesentest, Handlungsempfehlung, Artefakt) |
| Zentrale Begriffe bei erster Nennung definieren und konsistent verwenden                | Hypothesen/Erwartungen nur, wenn Theorie und Design sie tragen                           |

## 4. Literaturarbeit und Zitieren

Literatur ist nicht Kulisse, sondern **Begründungs- und Einordnungsinstrument**. Sie ist dann gut, wenn sie die eigene Argumentation schärft (Begriffe klärt, Erwartungen ableitet, Ergebnisse einordnet) – nicht, wenn sie viel ist.

| MUSS                                                                                    | SOLL                                                                                                 |
| --------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| Jede übernommene Idee/Definition/Zahl/These/Methode/Argumentation wird belegt           | Forschungsstand **synthetisieren** (Stränge/Positionen) statt Quellen nacheinander zu paraphrasieren |
| Direkte Zitate sparsam; bei direkten Zitaten immer Seiten-/Stellenangabe                | Quellenkritik berücksichtigen: Autorität, Begutachtung, Aktualität, Methodik, Interessenkonflikte    |
| Paraphrasen sind echte Umformulierungen (keine Patchwork-Synonyme)                      | Aktuelle Literatur nutzen, wenn das Feld dynamisch ist                                               |
| Primärquelle bevorzugen; Sekundärzitat nur bei Unzugänglichkeit und dann gekennzeichnet | Graue Literatur nur, wenn notwendig, und dann kritisch einordnen                                     |

## 5. Methodik

Methodik ist ein **Prüfprotokoll**. Wer den Methodikteil liest, muss verstehen, was genau getan wurde – so konkret, dass die Studie (innerhalb sinnvoller Grenzen) reproduzierbar wäre.

| MUSS                                                                                                                               | SOLL                                                                         |
| ---------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| Design benennen (z. B. Interview, Inhaltsanalyse, Survey, Experiment, Sekundärdaten, Fallstudie, Design Science)                   | Entscheidungssätze: „Methode X, weil …; Alternative Y verworfen, weil …“     |
| Daten/Material transparent: Auswahlkriterien, Umfang, Zeitraum, Ein-/Ausschluss                                                    | Reflexionsabschnitt (Bias/Rolle der Forschenden) bei qualitativen Designs    |
| Erhebung/Operationalisierung konkret: Instrumente, Leitfaden, Mess-/Kodierregeln                                                   | Robustheits-/Sensitivitätsüberlegungen, wenn quantitativ sinnvoll            |
| Auswertung konkret: Verfahren + Schritte + (ggf.) Software/Parameter                                                               | Reproduzierbare Artefakte in den Anhang: Code, Kategoriensystem, Instrumente |
| Gütekriterien explizit: qualitativ (Transparenz/Reflexivität/Triangulation) oder quantitativ (Validität/Reliabilität/Objektivität) | Präregistrierung, wenn in Ihrem Fach üblich                                  |

## 6. Wissenschaftlicher Sprachgebrauch

Sprache ist Teil der Methodik: Unklare Sprache produziert unprüfbare Ergebnisse. Wissenschaftliche Sprache ist **präzise**, **sachlich** und **funktional**.

| MUSS                                                                                   | SOLL                                                                                            |
| -------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| Verständlich, eindeutig, unmissverständlich, nüchtern, sachlich                        | Fremdwörter/Anglizismen nur, wenn präziser/geläufiger oder ohne sinnvolle deutsche Entsprechung |
| Grammatik, Zeichensetzung, Rechtschreibung korrekt und konsistent                      | Abkürzungen sparsam; bei erster Nennung erklären; bei vielen Abkürzungen Verzeichnis            |
| Hauptsache in den Hauptsatz; Nebensache in den Nebensatz                               | Ich-Form nur für Vorgehen/Struktur; Alternativen: Passiv, unpersönlich, deagentiviert           |
| Definierte Begriffe nicht umbenennen; abweichende Definitionen markieren und einordnen | Signposting gezielt nutzen („hingegen“, „daraus folgt“, „zusammengefasst“)                      |
| Vage Begriffe konkretisieren oder operationalisieren                                   | Nicht-diskriminierende Sprache konsistent umsetzen (eine Variante wählen und durchhalten)       |
| Wertungen nur als Sachurteil mit Kriterium/Evidenz (keine Meinungsmarker)              |                                                                                                 |

### 6.1 Füllwörter und Leerformeln

| MUSS                                                                                            | SOLL                                                                                        |
| ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| Füllwörter streichen, wenn kein Informationsgewinn entsteht                                     | Eigene „Füllwortliste“ aus Korrekturdurchläufen pflegen                                     |
| Leerwertungen vermeiden („interessant/wichtig/bedeutsam“) ohne Kriterium/Evidenz                | Streich-Test konsequent nutzen (Streichung darf die Aussage nicht verändern)                |
| Typische Füllwörter vermeiden: „eigentlich“, „irgendwie“, „quasi“, „sozusagen“, „gewissermaßen“ | Intensivierer nur, wenn quantifiziert oder operationalisiert („deutlich“, „erheblich“ etc.) |
| Pseudoargumente vermeiden: „wie jeder weiß“, „es ist allgemein bekannt, dass“                   | Floskeln ersetzen durch konkrete Aussage plus Evidenzanker                                  |
| Falsche Superlative vermeiden: „die einzigsten“, „am optimalsten“, „in keinster Weise“          |                                                                                             |

## 6.2 Wissenschaftliche Sprachbausteine

Sprachbausteine sind ein legitimes Werkzeug zur Formulierung. Sie sind **kein Inhalt**, sondern ein **Gerüst**, das mit projektspezifischer Substanz gefüllt werden muss.

| MUSS                                                                                                                    | SOLL                                                                                                    |
| ----------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| Bausteine/Patterns (auch aus dem Internet) frei nutzbar; **nicht zitierpflichtig**                                      | Bausteinbibliothek als Snippets pflegen (z. B. Abstract, Methodik, Diskussion)                          |
| Zitierpflicht betrifft **Inhalte**: Ideen, Definitionen, Daten, Argumente, Methodenentscheidungen, Klassifikationen     | Bausteine nach Funktion wählen (Definition, Abgrenzung, Begründung, Evidenz, Einordnung, Einschränkung) |
| Bausteine immer projektspezifisch füllen (Begriffe, Kriterien, Daten, Kontext)                                          | Übergänge markieren (Kontrast, Folge, Einschränkung, Zusammenfassung)                                   |
| Aussagekraft kalibrieren: stark („zeigt/belegt“) nur bei starker Evidenz; sonst „legt nahe/deutet darauf hin“           | Sprachliche Variation nutzen, ohne Bedeutungsverschiebung zu erzeugen                                   |
| Pro Absatz mindestens ein Substanzanker (Evidenz **oder** definitorische Präzisierung **oder** argumentative Ableitung) |                                                                                                         |

### 6.2.1 Bausteine (Deutsch)

| Funktion                  | Formulierungsbausteine (Beispiele)                                                                                                                |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Problem/Relevanz          | „Ausgangspunkt dieser Arbeit ist …“ / „Relevanz ergibt sich aus …, weil …“ / „Vor dem Hintergrund von … stellt sich die Frage, …“                 |
| Ziel/Beitrag              | „Ziel dieser Arbeit ist es, … zu untersuchen/zu erklären/zu evaluieren.“ / „Die Arbeit leistet einen Beitrag, indem …“ / „Im Fokus steht …, da …“ |
| Forschungsstand/Synthese  | „Die Literatur lässt sich in folgende Stränge gliedern: …“ / „Während A …, argumentieren B …“ / „Unklar bleibt bislang, ob/wie …“                 |
| Definition/Abgrenzung     | „Im Folgenden wird [Begriff] verstanden als … (vgl. [Quelle]).“ / „Davon abzugrenzen ist …, da …“                                                 |
| Operationalisierung       | „Operationalisiert wird [Konstrukt] über … (Mess-/Kodierregel: …).“                                                                               |
| Methodische Entscheidung  | „Zur Beantwortung der Fragestellung wird [Design] gewählt, weil …“ / „Die Auswertung erfolgt mittels … in den Schritten …“                        |
| Ergebnisdarstellung       | „Die Ergebnisse legen nahe, dass …“ / „Es ergibt sich ein Muster, wonach …“ / „Tabelle/Abbildung [X] verdeutlicht, dass …“                        |
| Einordnung/Interpretation | „Im Lichte von [Theorie/Quelle] lässt sich dies so erklären: …“ / „Im Vergleich zu [Quelle] deutet dies darauf hin, dass …“                       |
| Limitationen              | „Die Aussagekraft ist begrenzt durch …“ / „Aufgrund von … kann nicht ausgeschlossen werden, dass …“                                               |
| Ausblick                  | „Für zukünftige Forschung ergibt sich …“ / „Praktische Implikation: … unter der Bedingung, dass …“                                                |

### 6.2.2 Bausteine (Englisch)

| Function                  | Sentence starters (examples)                                                          |
| ------------------------- | ------------------------------------------------------------------------------------- |
| Aim/Question              | „This thesis examines …“ / „The study investigates whether/how …“                     |
| Method                    | „Methodologically, the study relies on …“ / „The analysis proceeds in three steps: …“ |
| Results                   | „The results indicate that …“ / „The findings suggest …“                              |
| Contribution/Implications | „The study contributes by …“ / „Implications for … are …“                             |
| Limitations               | „The findings are limited by …“ / „Results may not generalize beyond …“               |

## 7. Ergebnisse, Diskussion, Fazit

Dieses Kapitelbündel verhindert typische Bewertungsfehler: Ergebnisse werden belegt, Diskussionen begründet, und das Fazit synthetisiert ohne neue Analyse.

| MUSS                                                                     | SOLL                                                                           |
| ------------------------------------------------------------------------ | ------------------------------------------------------------------------------ |
| Ergebnisse: Befunde darstellen, nicht bewerten                           | Ergebnisse in sinnvollen Blöcken organisieren (Dimension/Variable/Codefamilie) |
| Tabellen/Abbildungen im Text ansteuern und auswerten                     | Pro Ergebnisblock: Kernaussage plus Evidenzanker                               |
| Null-/Gegenbefunde transparent berichten                                 | Visualisierungen nur, wenn sie eine Aussage tragen                             |
| Diskussion: Befunde erklären/einordnen und mit Literatur rückbinden      | Alternativdeutungen kurz prüfen (wo sinnvoll)                                  |
| Limitationen benennen (Design, Messung, Sample, Kontext, Interpretation) | Implikationen getrennt formulieren: Theorie vs. Praxis                         |
| Implikationen nur ableiten, wenn der Ableitungspfad nachvollziehbar ist  |                                                                                |

Das Fazit ist der synthetische Schlusspunkt: Es verdichtet die Argumentation zur klaren Antwort auf die Fragestellung. Prüfer*innen akzeptieren Schlussfolgerungen nur dann, wenn sie auf Befunde/Herleitungen im Hauptteil zurückgeführt werden können.

| MUSS                                                                              | SOLL                                                                     |
| --------------------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| Fragestellung explizit beantworten (keine Andeutungen)                            | Beitragstyp knapp einordnen (theoretisch/methodisch/empirisch/praktisch) |
| Kernaussagen bündeln (synthetisieren), nicht kapitelweise nacherzählen            | 1–2 Implikationen formulieren, strikt aus Ergebnissen ableitbar          |
| Jede Kernaussage ist im Hauptteil belegbar vorbereitet (Befund/Herleitung)        | 1–2 Anschlussfragen/Ausblick benennen (ohne neue Analyse)                |
| Keine neuen Daten, keine neuen Analyseschritte, keine neuen Argumentationsstränge | Limitationen knapp erinnern, falls für Interpretation entscheidend       |

## 8. Formeln, Symbole, Tabellen und Abbildungen

Formeln und Visualisierungen sind **Beweisstücke**: typografische Korrektheit, Definitionen und die Verarbeitung im Fließtext sind zwingend.

### 8.1 Formeln

| MUSS                                                                              | SOLL                                                                        |
| --------------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| Variablen/Symbole **kursiv**; Indizes/Deskriptoren **aufrecht** (normale Schrift) | Indizes in LaTeX als Roman markieren (z. B. mit `\mathrm{}` oder `\text{}`) |
| Jede verwendete Größe definieren (Symbol, Bedeutung, Einheit)                     | Symbolverzeichnis, wenn viele Formeln/Größen                                |
| Beispielnotation: `$F_{\mathrm{max}}$`, `$E_{\mathrm{kin}}$`, `$x_{\mathrm{i}}$`  | Notation über alle Kapitel konsistent halten                                |

### 8.2 Tabellen und Abbildungen

| MUSS                                                           | SOLL                                                                   |
| -------------------------------------------------------------- | ---------------------------------------------------------------------- |
| Fortlaufend nummerieren; präziser Titel/Caption; Quelle        | Caption so schreiben, dass das Element ohne Fließtext verständlich ist |
| Im Text referenzieren und auswerten (keine „stummen“ Elemente) | Pro Element eine Satz-Kernaussage („Die Abbildung zeigt …“)            |
| Keine dekorativen Grafiken                                     | Einheitliche Formatierung (Achsen, Einheiten, Skalierung)              |

## 9. Schlussprüfung

Die Endkontrolle ist ein Stresstest: Wenn die Arbeit in zehn Minuten nicht als prüfbar erscheint, wird sie im Detail kaum überzeugen.

| MUSS                                                                             | SOLL                                                                          |
| -------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| Einleitung und Fazit müssen allein die Frage-Antwort-Klammer verständlich machen | Laut lesen: syntaktische Holperstellen sind meist argumentative Holperstellen |
| Jede Kernaussage im Fazit ist im Hauptteil nachweisbar vorbereitet               | Letzte Konsistenzprüfung: Begriffe, Zitierstil, Abbildungsnummern, Notation   |
